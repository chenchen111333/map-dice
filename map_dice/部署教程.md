# 地图掷骰子 - 部署教程

本教程将手把手教你如何把网页部署到互联网，获得一个可长期使用的网址。选择以下任一方式即可。

---

## 三步快速开始

1. **部署网站**：推荐用 **Netlify**，打开 [netlify.com](https://www.netlify.com) 注册后，把 `map_dice` 文件夹**拖拽**到页面
2. **获得网址**：部署完成后会得到一个 `https://xxx.netlify.app` 的链接
3. **配置高德地图**：在高德控制台把该网址加入「允许的域名」，地图即可正常显示

下面为各平台的详细步骤。

---

## 部署后必做：配置高德地图域名

你的网页使用了高德地图，**部署完成后**需要在「高德开放平台」把新网址加入白名单，地图才能正常显示。

### 步骤（在获得部署网址之后）

1. 打开 [高德开放平台控制台](https://console.amap.com/)
2. 登录后，进入 **应用管理** → 找到你的应用
3. 点击 **添加** 或 **编辑**，在 **Web 端 (JS API)** 的 **安全密钥** 设置中找到 **允许的域名**
4. 添加你刚获得的部署网址，例如：
   - `https://xxx.netlify.app`（Netlify）
   - `https://xxx.vercel.app`（Vercel）
   - `https://xxx.pages.dev`（Cloudflare）
   - `https://用户名.github.io`（GitHub Pages）
   - 或临时填写 `*` 表示允许所有域名（仅测试用，不建议长期使用）
5. 保存

> 如果部署后地图是空白的，多半是这里没配置好，请按上述步骤添加域名。

## 方案一：Netlify 部署（推荐，最简单，支持拖拽）

Netlify 支持**直接拖拽文件夹**上传，无需 Git、无需命令行，最适合新手。

### 1. 注册账号

1. 打开 [https://www.netlify.com](https://www.netlify.com)
2. 点击 **Sign up**，用邮箱或 GitHub 注册

### 2. 拖拽部署

1. 登录后进入控制台（Dashboard）
2. 找到 **「Add new site」** 或 **「Sites」**，选择 **「Deploy manually」** 或 **「Import an existing project」**
3. 在出现的页面上，找到 **「Drag and drop your site output folder here」** 区域
4. 将你的 **`map_dice` 文件夹**从 Finder（Mac）或文件资源管理器（Windows）中**拖拽**到该区域
5. 等待 10–30 秒，部署完成后会自动生成一个网址，例如：`https://随机名称.netlify.app`

### 3. 获取长期网址

- 免费版提供：`https://你的站点名.netlify.app`
- 在 **Site settings** → **Domain management** 中可修改子域名为更好记的名称

## 方案二：Vercel 部署（需安装 Node.js）

适合已安装 Node.js 的用户，一条命令即可部署。

### 1. 安装 Node.js（如未安装）

1. 打开 [https://nodejs.org](https://nodejs.org)
2. 下载并安装 **LTS 版本**
3. 安装完成后，打开终端（Mac 为「终端」，Windows 为「命令提示符」）输入 `node -v`，如显示版本号则安装成功

### 2. 部署

1. 打开终端
2. 依次输入以下命令：

```bash
cd /Users/tal/Documents/map_dice
npx vercel
```

3. 首次运行会提示登录：浏览器会打开 Vercel 页面，用 GitHub 或邮箱登录
4. 按提示一路回车（使用默认设置即可）
5. 部署完成后会显示网址，例如：`https://map-dice-xxx.vercel.app`

### 3. 获取长期网址

- 该网址可长期使用
- 日后更新网页：修改 `index.html` 后，在同一目录再次执行 `npx vercel` 即可

## 方案三：Cloudflare Pages 部署（拖拽，国内访问快）

支持拖拽上传，且在国内访问速度通常较好。

### 1. 注册

1. 打开 [https://dash.cloudflare.com](https://dash.cloudflare.com)
2. 注册 Cloudflare 账号

### 2. 创建 Pages 项目

1. 登录后，左侧选择 **Workers & Pages**
2. 点击 **Create** → **Pages** → **Connect to Git** 或 **Upload assets**
3. 选择 **Upload assets**（直接上传）
4. 项目名称填：`map-dice`
5. 将 `map_dice` 文件夹中的**所有文件**拖入（或打包成 zip 后上传）
6. 点击 **Deploy**

### 3. 获取网址

部署完成后会得到：`https://map-dice.你的账号.pages.dev`

---

## 方案四：GitHub Pages 部署

适合已有 GitHub 账号、想学习 Git 的用户。

### 1. 注册 GitHub

1. 打开 [https://github.com](https://github.com)
2. 点击 **Sign up** 注册账号

### 2. 创建仓库

1. 登录后，点击右上角 **+** → **New repository**
2. **Repository name** 填：`map-dice`（可自定义）
3. 选择 **Public**
4. 点击 **Create repository**

### 3. 上传文件

1. 进入新建的仓库页面
2. 点击 **uploading an existing file**
3. 将 `index.html` 和 `README.md` 拖入（或整个 `map_dice` 里的文件）
4. 在底部点击 **Commit changes**

### 4. 开启 GitHub Pages

1. 进入仓库 → **Settings**
2. 左侧找到 **Pages**
3. 在 **Source** 中选择 **Deploy from a branch**
4. **Branch** 选 `main`，文件夹选 `/ (root)`
5. 点击 **Save**
6. 稍等 1–2 分钟，刷新页面会看到网址：`https://你的用户名.github.io/map-dice/`

## 常见问题

### 1. 地图是空白的

- 到 [高德开放平台](https://console.amap.com/) 检查应用设置
- 确认「允许的域名」中包含你的实际访问域名
- 确认 Key 和 安全密钥 已正确填写在 `index.html` 中

### 2. 网址太复杂 / 想用自定义域名

- **Vercel / Netlify / Cloudflare Pages**：在各自控制台都有「添加自定义域名」选项，可绑定你自己的域名（需先购买域名）
- 免费子域名（如 `.vercel.app`）已经可以长期使用

### 3. 想更新网页内容

- **Vercel / Netlify**：重新拖拽上传新版本，或重新执行 `npx vercel`
- **GitHub Pages**：在仓库中修改 `index.html` 等文件，提交后会自动重新部署

### 4. 在国内访问慢

- 可优先尝试 **Cloudflare Pages** 或 **Vercel**，国内访问速度相对较好
- 如使用 GitHub Pages 较慢，可考虑 Gitee Pages 等国内平台
